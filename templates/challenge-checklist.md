# Challenge Checklist

> 用于开题前、卡住时、复盘前的快速检查。  
> 目标：避免漏掉基础步骤，避免无头苍蝇式做题。

---

## A. 开题前检查

### 1. 我有没有完整读题？
- [ ] 看了题目标题
- [ ] 看了题面描述
- [ ] 看了题目分类
- [ ] 看了提示（Hints）
- [ ] 看了附件类型 / 下载了附件

### 2. 我有没有先判题型？
- [ ] Web
- [ ] Crypto
- [ ] Forensics
- [ ] Pwn
- [ ] Rev
- [ ] General Skills / Misc

### 3. 我有没有提取关键词？
- [ ] 关键词已写到 `notes.txt`
- [ ] 题面中的强提示词已记录
- [ ] 已写下初步猜测

---

## B. 做题过程检查

### 1. 我有没有先做低成本检查？
- [ ] 没有一上来就乱猜 payload
- [ ] 没有一上来就直接问答案
- [ ] 先做了最基本的信息收集

### 2. 我有没有记录过程？
- [ ] 关键命令记到了 `notes.txt`
- [ ] 关键输出 / 关键现象记下来了
- [ ] 失败尝试也有简单记录

### 3. 我有没有根据现象缩小范围？
- [ ] 不是机械乱试
- [ ] 每一步都有依据
- [ ] 能说清为什么换方向

---

## C. 分方向检查

### Forensics
- [ ] `file`
- [ ] `strings`
- [ ] `xxd`
- [ ] `exiftool`
- [ ] `pdfinfo`（如果是 PDF）
- [ ] `binwalk`
- [ ] `foremost`
- [ ] 是否检查了真实文件类型
- [ ] 是否检查了 metadata
- [ ] 是否考虑了隐藏文件 / 追加数据 / 嵌套文件

### Web
- [ ] 页面功能是否都点过
- [ ] 是否查看页面源码
- [ ] 是否查看 JS / 注释 / hidden 字段
- [ ] 是否用 Burp 抓包
- [ ] 是否观察 URL / POST 参数 / Cookie / Header
- [ ] 是否检查 `robots.txt`
- [ ] 是否考虑目录扫描 / 备份文件 / debug 路径
- [ ] 是否先确认“哪里可控”

### Crypto
- [ ] 是否先识别编码 / 密文格式
- [ ] 是否检查 Base64 / Hex / URL 编码
- [ ] 是否考虑常见古典密码
- [ ] 是否考虑 XOR
- [ ] 是否根据题面关键词判断方向
- [ ] 是否及时写 Python 脚本验证

### Pwn
- [ ] `file`
- [ ] `checksec`
- [ ] 程序是否运行过
- [ ] 是否观察输入点 / 输出点
- [ ] 是否有源码
- [ ] 是否 strings 过
- [ ] 是否考虑栈溢出 / 格式化字符串 / ret2win 等基础漏洞

### Rev
- [ ] `file`
- [ ] `strings`
- [ ] 是否运行观察程序行为
- [ ] 是否尝试 `ltrace` / `strace`
- [ ] 是否定位 main / 关键比较点
- [ ] 是否找到输入校验逻辑
- [ ] 是否分析了关键分支

---

## D. AI 使用检查

- [ ] 我是否先自己做了题型判断
- [ ] 我是否先自己做了基础检查
- [ ] 我问 AI 的是“下一步方法”而不是直接要答案
- [ ] 我向 AI 提供了现象和输出，而不是一句“这题怎么做”
- [ ] 即使 AI 给了提示，我最后也会自己复盘

---

## E. 收尾检查

### 1. 我有没有拿到 flag 之外的东西？
- [ ] 知道这题为什么这么做
- [ ] 知道题面提示在哪里
- [ ] 知道这题属于哪一类模板

### 2. 我有没有整理正式记录？
- [ ] 已完成 `solve.md`
- [ ] 已补充 `notes.txt`
- [ ] 已保存必要截图
- [ ] 已保存中间产物

### 3. 我有没有做知识沉淀？
- [ ] 已经思考哪些内容可以写入 `notes/`
- [ ] 已抽象出这类题的通用起手方法
- [ ] 已记录常见坑点

### 4. 我有没有提交同步？
- [ ] `git add .`
- [ ] `git commit -m "..."`
- [ ] `git push`
