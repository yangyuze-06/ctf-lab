# hashcrack - Writeup

## 题目信息

* **题目名称**：hashcrack
* **题目类型**：Cryptography
* **核心考点**：

  * 哈希类型识别
  * 弱密码哈希破解
  * 常见明文口令反查
  * `nc` 交互获取 flag

---

## 题目描述

题目给出一个远程服务：

```bash
nc verbal-sleep.picoctf.net 62342
```

连接后，服务端会依次给出若干个哈希值，要求我们输入对应的原始密码。如果输入正确，就会继续下一轮；全部答对后即可得到 flag。

题目的核心在于：**识别哈希算法，并还原其对应明文**。

---

## 解题思路

这题本质上是一道典型的 **弱哈希破解题**。

服务端返回的不是明文密码，而是密码经过哈希算法处理后的结果。由于这些密码都比较弱、比较常见，因此可以通过以下方式恢复原文：

1. **根据长度判断哈希类型**
2. **使用在线网站或本地工具查询对应明文**
3. **将得到的密码输入远程服务**
4. **连续完成多轮后拿到 flag**

---

## 题目分析

首先连接远程服务：

```bash
nc verbal-sleep.picoctf.net 62342
```

终端返回：

```text
Welcome!! Looking For the Secret?

We have identified a hash: 482c811da5d5b4bc6d497ffa98491e38
Enter the password for identified hash:
```

这里给出了第一组哈希值：

```text
482c811da5d5b4bc6d497ffa98491e38
```

观察其特征：

* 长度为 **32 位**
* 字符只包含十六进制字符 `0-9a-f`

因此可以判断它是 **MD5** 哈希。

将该哈希放入在线哈希反查网站，或者使用本地工具查询，可以得到原文为：

```text
password123
```

输入后，服务端继续给出第二组哈希：

```text
b7a875fc1ea228b9061041b7cec4bd3c52ab3ce3
```

其特征为：

* 长度 **40 位**

因此可以判断它是 **SHA1** 哈希。

查询后得到原文为：

```text
letmein
```

接着第三组哈希为：

```text
916e8c4f79b25028c9e467f1eb8eee6d6bbdff965f9928310ad30a8d88697745
```

其特征为：

* 长度 **64 位**

因此可以判断它是 **SHA256** 哈希。

查询后得到原文为：

```text
qwerty098
```

将三组密码依次输入后，即可得到最终 flag。

## 知识点总结

这题主要考查的是常见哈希算法的识别与弱密码反查能力。

### 1. 常见哈希长度

* **MD5**：32位
* **SHA1**：40位
* **SHA256**：64位

在做题时，看到哈希值先不要急着爆破，优先根据长度判断类型。

---

### 2. 常见破解方式

如果题目中的密码较弱、较常见，通常可以直接通过以下方式恢复：

* 在线哈希反查网站
* 本地工具，如：

  * `hashcat`
  * `john`
  * `hashid`

例如本地识别哈希类型：

```bash
hashid 482c811da5d5b4bc6d497ffa98491e38
```

例如使用 `hashcat` 进行字典攻击：

```bash
echo "482c811da5d5b4bc6d497ffa98491e38" > hash.txt
hashcat -m 0 hash.txt /usr/share/wordlists/rockyou.txt
```

---

### 3. 题目本质

这题并不是让我们“逆向还原哈希算法”，而是利用弱密码容易被查表或字典攻击命中的特点，找到其原始密码。

也就是说，哈希本身不可逆，但如果原密码太弱，就可以通过大量常见密码进行匹配，最终找到答案。
