# EVEN-RSA-CAN-BE-BROKEN??? - Writeup

## 题目信息

* **题目名称**：EVEN-RSA-CAN-BE-BROKEN???
* **题目类型**：Cryptography
* **核心考点**：

  * RSA 基础与参数关系（`N = p*q`）
  * 公钥模数批量审计
  * `gcd` 检测共享因子
  * 利用共享素因子恢复私钥并解密

---

## 题目描述

题目给出多组 RSA 参数（若干 `N` 与对应密文 `c`，公共指数 `e=65537`）。

目标是从这些数据中恢复明文，拿到 flag。

这题核心在于：**不同 RSA 模数之间错误地共享了因子**。

---

## 解题思路

这题本质是经典的 **Common Prime Factor Attack（共享素因子攻击）**。

如果两组 RSA 模数满足：

```text
N1 = p*q1
N2 = p*q2
```

那么：

```text
gcd(N1, N2) = p
```

一旦拿到 `p`，就能分解 `N1`，进而计算私钥并解密密文。

标准流程：

1. 两两计算所有 `N` 的 `gcd`
2. 找到 `gcd(Ni, Nj) > 1` 的组合
3. 分解目标模数 `N = p*q`
4. 计算 `phi = (p-1)(q-1)`
5. 计算私钥 `d = e^{-1} mod phi`
6. 解密 `m = c^d mod N`
7. 转字节得到 flag

---

## 题目分析

在 `crack.py` 里，脚本逻辑是：

```python
for i in range(len(Ns)):
    for j in range(i+1, len(Ns)):
        p = gcd(Ns[i], Ns[j])
        if p != 1:
            # 分解 N、求 d、解密
```

运行后发现：

```text
[+] Found common factor between N[0] and N[1]
p = 2
```

说明 `N[0]` 和 `N[1]` 都是偶数并共享因子 `2`。虽然在正常 RSA 中 `p` 应为大素数，但这里因为参数生成错误，直接暴露了可利用点。

随后针对 `N[0], C[0]`：

* `q = N[0] // p`
* `phi = (p-1)*(q-1)`
* `d = inverse(e, phi)`
* `m = pow(C[0], d, N[0])`

最后把 `m` 转成字节即可得到 flag。

## 解题过程

### 1. 读取题目给定参数

题目提供多组 `N`、`C`，以及公共指数：

```text
e = 65537
```

---

### 2. 批量检测模数共享因子

对所有 `N` 两两计算：

```python
p = gcd(Ns[i], Ns[j])
```

发现 `N[0]` 与 `N[1]` 的最大公约数为 `2`。

---

### 3. 利用共享因子恢复私钥

以第 0 组为目标：

```python
N = Ns[0]
c = Cs[0]
q = N // p
phi = (p - 1) * (q - 1)
d = pow(e, -1, phi)
```

---

### 4. 解密得到明文

```python
m = pow(c, d, N)
flag = m.to_bytes((m.bit_length() + 7) // 8, 'big')
```

---

### 5. 获得 flag

```text
picoCTF{tw0_1$_pr!m305af7255}
```

## 知识点总结

### 1. `gcd` 是 RSA 批量审计的高效入口

当拿到多组公钥模数时，优先做两两 `gcd` 检查，可快速发现共享因子漏洞。

---

### 2. 一旦因子泄露，RSA 私钥可直接恢复

知道 `p` 后：

* `q = N/p`
* `phi = (p-1)(q-1)`
* `d = e^{-1} mod phi`

即可完成私钥恢复并解密。

---

### 3. 题目本质

这题不是“硬算大数分解”，而是利用了 **密钥生成错误**（模数复用因子）。

RSA 安全性依赖于 `N` 难分解；当实现层面出现共享因子，系统会被直接击穿。

---

## 复盘

这题是 RSA 误配置场景的典型入门题。

可复用流程：

1. 收集多组 `(N, c, e)`
2. 批量 `gcd` 审计所有 `N`
3. 定位共享因子并分解目标模数
4. 计算 `d` 解密密文
5. 输出 flag

后续做同类题时，先跑 `gcd` 检查通常是性价比最高的第一步。
