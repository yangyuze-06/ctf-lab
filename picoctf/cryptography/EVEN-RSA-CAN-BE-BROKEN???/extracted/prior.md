# RSA 共因子攻击前置知识

## 一、这类题到底在考什么

这类题表面上是 RSA 题，但真正考的不是你去硬分解一个超大的 `N`，而是考你是否能发现 **密钥生成过程出了问题**。

题目的核心思想通常是：

- RSA 本身数学上没有被直接暴破
- 但如果生成 `p`、`q` 的过程不安全
- 那么攻击者可以从多个公钥之间找到漏洞
- 最终恢复私钥并解密密文

所以，这类题更像是：

> 不是在打败 RSA，而是在打败“错误实现的 RSA”

---

## 二、RSA 的基础概念

### 1. 两个大素数 `p` 和 `q`

RSA 会先生成两个大素数：

```text
p, q
````

然后计算：

```text
N = p * q
```

这里的 `N` 就是公钥的一部分。

---

### 2. 欧拉函数 `φ(N)`

对于 RSA 来说：

```text
φ(N) = (p - 1)(q - 1)
```

这个值在求私钥时非常重要。

---

### 3. 公钥 `e`

RSA 会选一个公开的指数 `e`，常见值是：

```text
e = 65537
```

这也是很多题里固定给你的值。

---

### 4. 私钥 `d`

私钥 `d` 满足：

```text
d ≡ e^(-1) mod φ(N)
```

也就是说，`d` 是 `e` 在模 `φ(N)` 意义下的逆元。

---

## 三、RSA 的加密和解密公式

### 1. 加密

如果明文是 `m`，那么密文 `c` 为：

```text
c = m^e mod N
```

Python 里常写成：

```python
c = pow(m, e, N)
```

---

### 2. 解密

拿到私钥 `d` 之后，可以解密：

```text
m = c^d mod N
```

Python 里是：

```python
m = pow(c, d, N)
```

---

## 四、为什么正常情况下 RSA 很难破

因为攻击者通常只知道：

* `N`
* `e`
* `c`

但不知道：

* `p`
* `q`
* `φ(N)`
* `d`

而如果 `N` 足够大，想从 `N = p * q` 中把 `p` 和 `q` 分解出来是非常困难的。

所以正常情况下，RSA 的安全性建立在：

> 大整数分解困难

---

## 五、这题为什么还能破

这类题的关键不在于暴力分解一个 `N`，而在于：

> 多个 `N` 之间可能共享同一个素数因子

例如：

```text
N1 = p * q1
N2 = p * q2
```

如果两个公钥都重复使用了同一个 `p`，那么就会出现严重漏洞。

---

## 六、最大公约数 gcd

### 1. 什么是 gcd

`gcd(a, b)` 表示 `a` 和 `b` 的最大公约数，也就是同时整除这两个数的最大整数。

例如：

```text
gcd(15, 21) = 3
```

因为 3 同时整除 15 和 21。

---

### 2. Python 中怎么求 gcd

```python
from math import gcd

print(gcd(15, 21))  # 3
```

---

## 七、共因子攻击的核心原理

假设：

```text
N1 = p * q1
N2 = p * q2
```

那么：

```text
gcd(N1, N2) = p
```

因为：

* `p` 是 `N1` 的因子
* `p` 也是 `N2` 的因子
* 而且通常 `q1` 和 `q2` 不同

所以一旦两个 `N` 共享了同一个素数，就可以直接通过 `gcd` 把这个素数找出来。

这就叫：

# 共因子攻击（Common Factor Attack）

---

## 八、拿到 `p` 之后怎么继续

一旦求出：

```text
p = gcd(N1, N2)
```

就可以继续算出：

```text
q = N / p
```

然后得到：

```text
φ(N) = (p - 1)(q - 1)
```

接着求出私钥：

```text
d = e^(-1) mod φ(N)
```

最后解密：

```text
m = c^d mod N
```

于是整个 RSA 就被恢复了。

---

## 九、Python 里会用到的几个核心函数

### 1. `gcd`

```python
from math import gcd
g = gcd(N1, N2)
```

作用：求两个整数的最大公约数。

---

### 2. `pow(a, b, mod)`

```python
pow(m, e, N)
```

作用：快速计算：

```text
a^b mod mod
```

在 RSA 加解密里非常常见。

---

### 3. `inverse`

如果使用 `pycryptodome`：

```python
from Crypto.Util.number import inverse
d = inverse(e, phi)
```

作用：求模逆元。

---

### 4. 字节串和整数转换

```python
from Crypto.Util.number import bytes_to_long, long_to_bytes
```

作用：

* `bytes_to_long(b"abc")`：把字节转成整数
* `long_to_bytes(123456)`：把整数转回字节

因为 RSA 实际上处理的是整数，不是直接处理字符串。

## 十、这类题常见解题流程

### 第一步：多次拿到服务返回的 `N`

题目如果提示你：

```text
Try comparing N across multiple requests
```

意思通常是：

* 多连几次服务
* 每次记录新的 `N`
* 检查不同 `N` 之间是否有公共因子

---

### 第二步：两两求 gcd

把拿到的多个 `N` 放在一起，两两计算：

```python
gcd(N[i], N[j])
```

如果结果不是 1，就说明它们共享因子。

---

### 第三步：恢复 `p` 和 `q`

一旦发现：

```text
p = gcd(N1, N2)
```

那么：

```text
q = N1 / p
```

---

### 第四步：恢复私钥

根据：

```text
φ(N) = (p - 1)(q - 1)
d = inverse(e, φ(N))
```

算出私钥。

---

### 第五步：解密密文

```python
m = pow(c, d, N)
```

然后把整数转回字节串即可得到 flag。

---

## 十一、你需要掌握到什么程度

对于当前阶段，不需要一开始就学很深的数论，你只要先掌握下面这些就够做这类题：

### 必须会的

* 知道 RSA 的基本结构：`N = p*q`
* 知道加密解密公式
* 知道 `φ(N) = (p-1)(q-1)`
* 知道 `d` 是 `e` 的逆元
* 会用 `gcd`
* 知道如果两个 `N` 共用一个素数，就可以通过 `gcd` 爆出来

### 暂时不用会的

下面这些现在不用急着学：

* Wiener Attack
* Coppersmith Attack
* Boneh-Durfee
* 格攻击
* 高等数论证明

你现在只需要把“共因子攻击”这个模型吃透，就已经足够了。

## 十二、最小代码模板

下面是一个共因子攻击最小模板：

```python
from math import gcd
from Crypto.Util.number import inverse, long_to_bytes

N1 = ...
N2 = ...
e = 65537
c = ...

p = gcd(N1, N2)
q = N1 // p

phi = (p - 1) * (q - 1)
d = inverse(e, phi)

m = pow(c, d, N1)
print(long_to_bytes(m))
```

---

## 十三、这类题的本质总结

这类题最重要的认识不是“RSA 很脆弱”，而是：

> RSA 很强，但错误的密钥生成会让它瞬间失效

所以题目的攻击点是：

* 随机数生成不安全
* 质数生成重复
* 多次请求复用了某个素数
* 最终导致多个 `N` 存在公共因子

攻击者再利用 `gcd` 恢复因子，重建私钥，完成解密。

---

## 十四、一句话记忆版

可以记住这一句：

> 如果两个 RSA 模数 `N1` 和 `N2` 共享素数因子，那么 `gcd(N1, N2)` 直接泄露这个因子。
